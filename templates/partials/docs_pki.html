Certificate Lifecycle

OpenCPO monitors all certificate expiry dates and surfaces warnings on the PKI page. The recommended workflow:

  1. Issue — PKI page → Issue Certificate → select type, enter subject details, set validity period. Certificate is signed immediately by the appropriate sub-CA.
  2. Install — Download the signed certificate + private key bundle. For chargers, push via OCPP CertificateSigned or install manually per the charger's UI. For contract certs, deliver to the vehicle via the Charge App provisioning flow.
  3. Monitor — PKI dashboard shows days-to-expiry for all issued certificates. Set up alerts for <60 days.
  4. Rotate — Issue a new certificate before the old one expires. For chargers, use the InstallCertificate + CertificateSigned OCPP flow for zero-downtime rotation.
  5. Revoke — If a device is stolen or a key is compromised, revoke immediately. OpenCPO updates the CRL (Certificate Revocation List) and OCSP responder instantly. Connected chargers will reject revoked certs on the next TLS handshake.
Security note: Never share private keys between certificates. Each charger gets its own key pair. If a charger is physically compromised, revoke only that charger's certificate — not the entire CA chain. This is why per-device certificates matter.
06 ISO 15118 — Plug & Charge ISO 15118-2 Advanced

Plug & Charge (PnC) is the most significant UX advancement in EV charging since the Type 2 connector. The driver plugs in. The car and charger exchange certificates over ISO 15118-2. Charging starts. No app, no card, no PIN. It works the same way HTTPS authenticates your bank — the certificate carries the identity.

Most CPOs don't support Plug & Charge because it requires a functioning PKI, ISO 15118-capable chargers, and careful implementation of the certificate provisioning flow. OpenCPO provides all of this.

Plug & Charge — Session Sequence

Requirements

Charger Requirements
  • ISO 15118-2 support (check charger specs — not all OCPP chargers have this)
  • OCPP 2.0.1 (preferred) or OCPP 1.6j with ISO 15118 extension
  • Onboard SECC hardware module for TLS termination at the cable level
  • Example: HG Power ENC-DCL120B (120 kW DC, dual CCS2, ISO 15118-2 certified)
Vehicle Requirements
  • ISO 15118-2 support (most newer BEVs from 2021+ have this)
  • Contract certificate provisioned in the vehicle's secure element
  • Confirmed support: Porsche Taycan, Audi e-tron GT, VW ID. series, BMW iX, Mercedes EQ series

Setup Checklist

  1. Verify charger supports ISO 15118-2 — check the spec sheet, not just "smart charging" marketing language
  2. Issue a SECC certificate from PKI → Issue Certificate → type: SECC
  3. Install the certificate on the charger using OCPP InstallCertificate or the charger's configuration interface
  4. Driver provisions their contract certificate: open the Charge App → Profile → Plug & Charge → Request Contract Cert
  5. OpenCPO issues a contract certificate signed by the Contract Sub-CA and returns it to the app
  6. App delivers certificate to vehicle via OTA or manual installation (vehicle-dependent)
  7. Test: driver plugs into the ISO 15118-capable charger — session should start without any app interaction
Why most CPOs don't offer this: Plug & Charge requires a functioning PKI, ISO 15118 expertise, a certificate provisioning flow, and chargers that actually implement the standard correctly. The complexity is real. OpenCPO provides all of it out of the box — the only variable is your charger hardware.
