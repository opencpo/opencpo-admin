Charger Site (Raspberry Pi gateway)

  1. Get a Raspberry Pi 4 or any Linux SBC. Connect it to the local network at the charging location.
  2. Use the "Add New Site" wizard above to generate a join command. Run it on the Pi.
  3. Once connected, configure each charger's OCPP WebSocket URL to point to this server's Tailscale IP address — for example: ws://100.64.0.1:9100/{{ "charger-id" }}
  4. The charger will now communicate with the OCPP Core through the encrypted tunnel. No firewall changes needed.

Additional Server Node

  1. Install OCPP Core on the additional server. Configure it with the same database connection string.
  2. Join it to the tailnet using the "Server Node" option in the wizard above.
  3. Update your load balancer or DNS to distribute charger connections across both Core instances.
  4. Both nodes share state via the database — charger sessions are visible from either node.

Edge Proxy (Caddy / Kong)

  1. Deploy Caddy or nginx on a machine near the chargers. Join it to the tailnet using the wizard.
  2. Configure the proxy to forward OCPP WebSocket connections to the Core's Tailscale IP. Example Caddyfile:
  3. :9100 { reverse_proxy 100.64.0.1:9100 }
  4. Point chargers at the proxy's local IP. Traffic is forwarded through the encrypted tunnel to Core.
